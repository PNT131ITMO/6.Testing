# Лабораторная работа №1 — Тестирование программного обеспечения (Variant: 374807)

## Описание
Проект содержит выполнение лабораторной работы №1 по дисциплине **«Тестирование программного обеспечения»**.

**Вариант:** `374807`  
**Задания варианта:**
1. Функция `arcsin(x)` — тестирование разложения в степенной ряд.
2. Программный модуль сортировки массива методом **блочной сортировки (Bucket Sort)**.
3. Доменная модель по тексту (Глава 8):  
   > «Зажужжал мотор. Тоненький свист перерос в рев воздуха, вырывающегося в черную пустоту, усеянную невероятно яркими светящимися точками. Форд и Артур вылетели в открытый космос, как конфетти из хлопушки.»

---

## Цели лабораторной работы
1. Провести **модульное тестирование** разложения функции в степенной ряд и выбрать достаточное тестовое покрытие.
2. Провести **модульное тестирование алгоритма**: выбрать характерные точки внутри алгоритма, для самостоятельно предложенных наборов входных данных записать последовательность попаданий в характерные точки и сравнить с эталонной.
3. Сформировать **доменную модель** для заданного текста и разработать тестовое покрытие для доменной модели.

---

## Структура проекта
Проект организован как multi-module Gradle:

```
Lab1/
  part1/   # arcsin(x) (степенной ряд) + unit tests
  part2/   # bucket sort + тестирование по "характерным точкам"
  part3/   # доменная модель текста + unit tests
```

В каждом модуле:
- `src/main/java` — исходный код
- `src/test/java` — тесты (JUnit 5)
- `build.gradle` — настройки сборки

## Часть 1 — arcsin(x)
**Суть:** реализовать `arcsin(x)` через степенной ряд (с обработкой граничных значений) и покрыть модульными тестами.  
Тесты включают:
- проверку области определения и невалидных входов,
- граничные значения `x = -1, 0, 1`,
- сравнение с `Math.asin(x)` на наборе точек,
- property-based проверки (нечётность, монотонность),
- значения близкие к `±1` (проверка устойчивости/сходимости).

---

## Часть 2 — Bucket Sort
**Суть:** протестировать алгоритм блочной сортировки через “характерные точки” (ветвления/этапы):
- проверка валидации входных данных,
- распределение элементов по бакетам,
- сортировка внутри бакетов,
- сбор результирующего массива.

Для каждого набора данных фиксируется последовательность прохождения характерных точек и сравнивается с эталоном.

---

## Часть 3 — Доменная модель
**Суть:** построить объектную модель (entities + events + commands/controllers) по тексту и протестировать:
- корректность инвариантов (валидность модели),
- корректность переходов состояния (двигатель: buzz→whistle→roar),
- корректность действий (открытие шлюза, “вылет” персонажей в космос),
- логирование событий (DomainEvent) и пред-/постусловия.
